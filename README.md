# RAG-бот для Telegram

Telegram-бот с RAG (Retrieval-Augmented Generation): кидай файлы в папку `docs/` → они индексируются → общайся с их содержимым через Telegram. Работает полностью локально через Ollama.

## Требования

- Docker + Docker Compose
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))

## Быстрый старт

**1. Настройка окружения**
```bash
cp .env.example .env
# Открой .env и вставь TELEGRAM_TOKEN
```

**2. Запуск**
```bash
docker compose up --build -d
```

**3. Скачать модели (один раз)**
```bash
docker compose exec ollama ollama pull nomic-embed-text
docker compose exec ollama ollama pull phi3:mini
```

**4. Перезапустить бота**
```bash
docker compose up -d bot
```

## Использование

Положи файлы (PDF, DOCX, TXT, MD) в папку `docs/` и отправь боту `/reindex`.  
После этого просто задавай вопросы по содержимому документов.

**Команды бота:**
- `/start` — начало работы
- `/help` — справка
- `/reindex` — переиндексировать папку `docs/`

## Настройки (.env)

| Переменная | Описание | По умолчанию |
|---|---|---|
| `TELEGRAM_TOKEN` | Токен бота от BotFather | — |
| `OLLAMA_MODEL` | Модель LLM | `phi3:mini` |
| `OLLAMA_EMBED_MODEL` | Модель эмбеддингов | `nomic-embed-text` |
| `TOP_K` | Кол-во чанков для контекста | `5` |
| `HISTORY_LENGTH` | Кол-во сообщений в истории | `10` |
| `CHUNK_SIZE` | Размер чанка при индексации | `300` |
| `CHUNK_OVERLAP` | Перекрытие чанков | `50` |

## Поддерживаемые локальные модели

Все модели запускаются через Ollama в CPU-режиме. Скорость указана ориентировочно для Intel N150.

| Модель | Параметры | RAM | Скорость (N150) | Примечание |
|---|---|---|---|---|
| `qwen2.5:1.5b` | 1.5B | ~1 ГБ | ~20–35 сек/ответ | Минимальные требования, базовое качество |
| `gemma2:2b` | 2B | ~1.6 ГБ | ~30–50 сек/ответ | Лучше qwen2.5:1.5b по качеству рассуждений |
| `llama3.2:3b` | 3B | ~1.9 ГБ | ~40–70 сек/ответ | Хорош для многоязычных документов |
| `qwen2.5:3b` | 3B | ~1.9 ГБ | ~40–70 сек/ответ | Сильный для структурированных текстов |
| `phi3:mini` ✅ | 3.8B | ~2.3 ГБ | ~50–90 сек/ответ | **Рекомендуется** — лучшее следование инструкциям |
| `phi4-mini` | 3.8B | ~2.5 ГБ | ~50–90 сек/ответ | Новее phi3:mini, чуть точнее |
| `mistral:7b` | 7B | ~4.1 ГБ | ~2–4 мин/ответ | Высокое качество, но медленно на CPU |

> **Минимум для запуска:** 4 ГБ RAM (модель + ОС + эмбеддинги).  
> На машинах с 8+ ГБ RAM комфортно работают модели до 3.8B.  
> Модели 7B и выше на CPU дают ответ за несколько минут — используй только если скорость не критична.

Сменить модель: поменяй `OLLAMA_MODEL` в `.env`, скачай нужную и перезапусти бота:
```bash
docker compose exec ollama ollama pull phi4-mini
# поменяй OLLAMA_MODEL=phi4-mini в .env
docker compose up -d bot
```

## Стек

- **Бот:** aiogram 3.x
- **RAG:** LangChain + ChromaDB
- **LLM:** Ollama (CPU-режим, без GPU)
- **История:** SQLite
